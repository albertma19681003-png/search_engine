**Мощная поисковая система для корпоративного портала компании "АйТиБокс"**

Проект реализует полнофункциональный поисковый движок с инвертированным индексом, способный быстро находить релевантные документы по пользовательским запросам. Разработан профессиональной командой программистов для обеспечения эффективного поиска по корпоративной документации.

## 🚀 Особенности

- **⚡ Быстрый поиск** - Инвертированный индекс для мгновенного поиска
- **📊 Интеллектуальное ранжирование** - Расчет релевантности с нормализацией
- **🔧 Гибкая конфигурация** - JSON-файлы для настройки без перекомпиляции
- **🧪 Полное тестирование** - Комплексные unit-тесты с Google Test
- **🔄 Многопоточность** - Параллельная индексация документов
- **📁 Поддержка форматов** - Работа с текстовыми документами
- **🎯 Точные результаты** - Уникализация слов запроса и сортировка по релевантности

## 🛠️ Установка и сборка

### Предварительные требования

- **C++ Compiler** с поддержкой C++17 (GCC/Clang/MSVC)
- **CMake** версии 3.16 или выше
- **Git** для клонирования репозитория


# Сборка
cmake --build .

# Запуск тестов
ctest
Сборка в CLion
Откройте проект в CLion


1. Настройка конфигурации
   
Создайте файл resources/config.json:


json

{

    "config": {
    
        "name": "ITBoxSearchEngine",
        
        "version": "3.23",
        
        "max_responses": 5
        
    },
    
    "files": [
    
        "resources/file001.txt",
        "resources/file002.txt",
        "resources/file003.txt",
        "resources/file004.txt"
    ]
}
Обязательные поля:

config - должен содержать данные (не может быть пустым {})

version - должна быть "3.23"

max_responses - максимальное количество результатов на запрос

files - пути к индексируемым файлам (хотя бы один файл)

2. Подготовка документов
   
Разместите текстовые файлы в папке resources/. Каждый файл должен содержать текст для индексации.


Пример документа (resources/file001.txt):


text
milk milk milk milk water water water

3. Создание запросов
   
Создайте файл resources/requests.json:


json

{

    "requests": [
    
        "milk water",
        
        "sugar",
        
        "london is the capital",
        
        "cappuccino"
        
    ]
    
}

4. Запуск поиска
   
 CLion
 
Run → Run 'search_engine'

5. Получение результатов
   
Результаты сохраняются в файл resources/answers.json:


json

{

    "answers": {
    
        "request001": {
        
            "result": "true",
            
            "relevance": [
            
                {"docid": 0, "rank": 1.0},
                
                {"docid": 1, "rank": 0.7},
                
                {"docid": 2, "rank": 0.3}
                
            ]
            
        },
        
        "request002": {
        
            "result": "false"
            
        }
        
    }
    
}

🧪 Тестирование
Проект включает комплексные unit-тесты:

bash
# Запуск всех тестов
./run_tests

# Запуск конкретной тестовой группы
./run_tests --gtest_filter="TestCaseInvertedIndex.*"

# Подробный вывод

./run_tests --gtest_color=yes

Тестовые сценарии

✅ Индексация документов - проверка построения инвертированного индекса

✅ Поиск по одному слову - базовый функционал поиска

✅ Многопоточная индексация - параллельная обработка документов

✅ Расчет релевантности - корректность ранжирования результатов

✅ Обработка граничных случаев - пустые запросы, отсутствующие слова

✅ Сортировка результатов - по релевантности и doc_id

🔧 Алгоритм работы

Фаза 1: Индексация

Загрузка документов - чтение текстовых файлов из config.json

Построение индекса - создание инвертированного индекса

Многопоточная обработка - параллельная индексация документов

Сортировка по doc_id - гарантия стабильных результатов

Фаза 2: Поиск
Разбор запроса - разделение на уникальные слова

Поиск документов - нахождение документов по индексу

Расчет релевантности - абсолютная и относительная релевантность

Сортировка - по убыванию релевантности, при равенстве - по возрастанию doc_id

Фаза 3: Формирование ответа
Применение лимита - ограничение результатов через max_responses

Сортировка doc_id - в массиве relevance по возрастанию

Сериализация - преобразование в JSON-формат

Сохранение - запись результатов в answers.json
